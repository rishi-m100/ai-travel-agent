📖 Scoring Methodology
▼

How Scores Are Calculated

1. Constraint Satisfaction (CS) - 0.0 to 1.0

Formula: CS = met_constraints / total_constraints

Measures how well the agent satisfies task requirements. Each task has success criteria (e.g., "must_have_flight", "correct_route", "is_cheapest"). The benchmark:

  • Counts total criteria defined in the task
  • Queries the database to verify each criterion
  • For "is_cheapest": Runs SELECT MIN(price) SQL query to find optimal price
  • Awards 1 point per satisfied constraint

Example: 4 out of 5 constraints met → CS = 0.8

2. Budget Efficiency (BE) - 0.0 to 1.0

Formula: BE = 1.0 - ((agent_cost - optimal_cost) / budget)

Measures cost-effectiveness of the agent's solution:

  • agent_cost: Total cost of the agent's selected items
  • optimal_cost: Cheapest possible cost (from SQL MIN query)
  • budget: Currently hardcoded as max(optimal_cost × 10, $5000)
  • If agent exceeds budget → BE = 0.0
  • Score clamped between 0.0 and 1.0

Example: Optimal: $200, Agent: $300, Budget: $2000 → BE = 1.0 - ((300-200)/2000) = 0.95

3. Logistics Score (LS) - 0.0 to 1.0

Formula: Starts at 1.0, deducts points for violations

Measures logical consistency and catches errors:

  • -0.3: Agent error occurred
  • -0.2: Check-out date not after check-in date
  • -0.1: Hotel nights mismatch (calculated vs reported)
  • -0.1: Invalid date format
  • Score clamped to minimum of 0.0

Example: No violations → LS = 1.0; Date error + nights mismatch → LS = 0.8

4. Evaluation Score (Composite)

Formula: S = (0.5 × CS) + (0.3 × BE) + (0.2 × LS)

Weighted combination of all three metrics:

  • 50% weight: Constraint Satisfaction (most important)
  • 30% weight: Budget Efficiency
  • 20% weight: Logistics Score

Pass Threshold: ≥ 0.75

Example: CS=0.8, BE=0.95, LS=1.0 → S = (0.5×0.8) + (0.3×0.95) + (0.2×1.0) = 0.885 ✓ PASS

Note: Each task is run multiple times (default: 3 runs) and scores are averaged to account for variability in agent performance.
